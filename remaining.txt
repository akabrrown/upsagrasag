## UNIVERSAL CODE RULES

### Always
- Complete every function — no stubs, no empty bodies
- Type everything explicitly — no implicit types, no `any`
- Handle every state — loading, empty, error, success
- Mobile-first — every UI component responsive by default
- Validate all inputs — on both client and server
- Extract shared logic — no copy-pasted code blocks
- Use named constants — no magic numbers or hardcoded strings
- Semantic HTML — proper elements, ARIA labels, keyboard nav

### Never
- Leave a TODO or placeholder comment in delivered code
- Store secrets in client-visible code or environment variables
- Skip error handling on async operations
- Use inline styles when a utility class exists
- Fetch data inside interactive/client components — pass as props
- Trust user input without validation
- Write a UI component without all three states: loading, empty, error

## FEATURE BUILD ORDER
When I ask for any feature, always deliver in this sequence:

1. Database — schema, migrations, RLS policies
2. Types — TypeScript interfaces and/or Zod schemas
3. Data layer — server-side query functions
4. API — route handlers with full validation and auth checks
5. UI — complete components, all states wired
6. Integration — data → API → UI connected end to end

Never skip a layer. Never deliver a layer that depends
on a layer you haven't built yet.

## COMPONENT RULES
- Default to Server Components; add interactivity only where needed
- Forms: always controlled via React Hook Form + Zod
- Images: always optimized — next/image or framework equivalent
- Never fetch inside a client/interactive component — use a parent
- Shared primitives go in /components/ui/; never modify them directly
- Feature components go in /components/{feature}/

---

## AUTH & SECURITY RULES
- Protect private routes at the middleware/server level
- Validate sessions server-side on every protected API call
- Return 401 from API when auth check fails — never silently proceed
- Never expose service keys, webhook secrets, or admin tokens to client
- Validate and sanitize all user-submitted data before DB writes
- Rate-limit any public-facing mutation endpoint

---

## FILE & FOLDER CONVENTIONS
Adapt naming to whatever framework I'm using, but always:

- Routes / pages in their framework-standard location
- Shared components in /components/
- Business logic and queries in /lib/ or /server/
- Type definitions in /types/
- Constants and config in /lib/config or /constants/
- Utility functions in /lib/utils/
- Environment variables documented in .env.example
